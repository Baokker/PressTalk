# 开发路线图

## Phase 2 — 智能整理模式 ✅ 已实现

### 2.1 基础润色（LLM 后处理）✅

转录完成后，文本交给 DeepSeek 做轻量后处理：去口头禅、修标点、处理自我纠正。

- 触发方式：右 Command 智能整理模式（普通右 Option 模式不经 LLM）
- 模型：DeepSeek `deepseek-chat`

### 2.2 智能整理模式（右 Command）✅

- 触发方式：按住右 Command 说话，松开后转录 + 智能整理
- 输出样式：去口头禅、处理自我纠正；仅当明确列举多个点时才输出列表

## Phase 3 — 准确率与速度提升

### 3.1 豆包 ASR 后端 ✅ 已实现

使用火山引擎豆包 ASR（bigmodel_nostream），即豆包输入法背后的引擎，准确率高。
凭 `VOLC_APP_ID` / `VOLC_ACCESS_KEY` 鉴权，无需其他配置。

### 3.2 降低端到端延迟

当前瓶颈分析：
- WebSocket 建连 + TLS 握手：~200–400ms
- 服务端推理（bigmodel_nostream）：~500ms–1s
- 文字注入：~50ms

优化方向：
- **连接预热**：程序启动时后台建立一次握手，预热 TCP/TLS 和 DNS 缓存，后续建连降至 ~50ms ✅ 已实现
- **流式逐字注入**：边说话边实时输出文字（类豆包输入法）— 需切换到流式 ASR 端点并重构注入逻辑，技术难度高，列入低优先级

## Phase 4 — GUI 与产品化

### 4.1 菜单栏应用（Menu Bar App）✅ 已实现

用 `rumps` 将程序改造为常驻菜单栏后台应用，入口为 `app.py`。

功能：
- 菜单栏图标实时显示状态（🎙 待机 / 🔴 录音中 / ⏳ 转录中 / ✓ 完成）
- 菜单项：今日转录次数、智能整理开关、退出
- `python3 main.py` 命令行模式仍可用

待完善：
- 开机自启（`launchd` plist）

### 4.2 设置界面

轻量设置页，替代手动编辑 `.env`：
- API Key 填写与验证
- 触发键自定义
- 智能整理开关（当前已在菜单中实现）

### 4.3 打包分发（`setup.py`）✅ 配置已就绪

```bash
pip install py2app
python setup.py py2app   # 输出 dist/VoiceInput.app
```

将 `dist/VoiceInput.app` 拖入 Applications 即可，无需安装 Python。
需将 `.env` 放在 `~/.voiceinput.env` 或打包时内嵌（待定）。

## Phase 5 — 体验细节

### 5.1 自定义触发键

当前触发键硬编码为右 Option / 右 Command，不够灵活。计划支持通过配置文件指定任意修饰键组合。

### 5.2 历史记录

本地记录每次转录的文本和时间戳，方便查找和复用之前说过的内容。

### 5.3 自定义词汇表

允许用户添加常用词、专业术语，提升特定领域（如代码、人名、品牌）的识别准确率（豆包 API 支持热词功能）。

### 5.4 多语言模式

当前默认中文优先。后续可支持语言快速切换，或自动语言检测模式。

### 5.5 实时状态反馈

当前在语音输入结束后，用户面对的是一个"黑盒"——不知道程序是在正常处理还是卡住了。对标产品的做法：豆包输入法实时流式输出字符；Typeless 显示 "thinking" 进度条，结束后再注入文字。

计划为本工具补充类似的状态提示，候选方案：

| 方案 | 实现难度 | 效果 |
|------|----------|------|
| **系统通知（`osascript` / `pync`）** | 低 | 录音结束时弹出 "转录中…"，完成后弹出结果预览 |
| **菜单栏图标动画** | 中（依赖 Phase 4.1）| 图标实时切换：待机 → 录音中 → 转录中 → 完成 |
| **浮动状态窗口** | 中 | 屏幕角落显示小悬浮框，实时更新状态文字和进度条 |

近期可先用系统通知作为轻量过渡方案，GUI 阶段再升级为菜单栏图标动画。
