# 开发路线图

## Phase 2 — 智能整理模式

**目标：** 接入大模型，将口语化转录稿自动整理为结构化文字。

### 2.1 基础润色（LLM 后处理）

对标豆包输入法的「大模型润色」环节：转录完成后，将文本交给 LLM 做一轮轻量后处理——去除口头禅（"就是那个"、"然后然后"）、修正标点、适当断句。

- 触发方式：右 Command 智能整理模式触发（普通模式不经 LLM）
- 模型：DeepSeek `deepseek-chat`（当前 `.env` 中已有 Key，尚未接入）
- 延迟目标：< 1 秒（DeepSeek 响应极快，输入量小）

### 2.2 智能整理模式（右 Command）

更激进的整理：将散乱口述转为有序列表、分段说明，适合整理思路、记笔记场景。

- 触发方式：按住右 Command 说话，松开后转录 + 智能整理
- 模型：DeepSeek `deepseek-chat`
- 输出样式：去口头禅、处理自我纠正；仅当明确列举多个点时才输出列表

## Phase 3 — 准确率与速度提升

### 3.1 解决 Whisper 英文自动翻译问题

当前 faster-whisper 在中文语境下偶尔会将夹杂的英文词汇翻译成中文。

解决方案：
- 转录时传入 `task="transcribe"`（明确不做翻译）
- 同时设 `language="zh"` 但允许英文词汇保留（Whisper 默认行为应已支持，需验证）

### 3.2 STT 后端扩展 ✅ 已实现（volcengine）

目前讯飞的识别质量优于本地 Whisper，但仍与豆包输入法存在差距。可选方向：

| 方案 | 说明 | 状态 |
|------|------|------|
| **豆包 / 火山引擎 STT** | Seed-ASR 2.0，即豆包输入法背后引擎；支持"二遍识别"（enable_nonstream），先实时出字再高精度修正 | ✅ 已接入，`STT_BACKEND=volcengine` |
| **阿里云 FunASR** | 开源，支持本地部署，中文效果优秀，可完全离线 | 待评估 |
| **讯飞极速录音转写大模型** | 讯飞新版极速模式，延迟更低 | 待评估 |

切换方式：`.env` 中设置 `STT_BACKEND=volcengine` 并填写 `VOLC_APP_ID` / `VOLC_ACCESS_KEY`。

### 3.3 降低端到端延迟

当前瓶颈分析（讯飞模式）：
- WebSocket 建连：~200ms
- 音频发送：接近 0（已去除 sleep）
- 服务端转录：~500ms–1s
- 文字注入：~250ms

优化方向：
- **连接复用**：程序启动时预建 WebSocket 长连接，录音结束直接发送，省去每次建连时间
- **流式注入**：转录结果边返回边注入，而非等全部完成再一次性粘贴（需要重构注入逻辑）

## Phase 4 — GUI 与产品化

### 4.1 菜单栏应用（Menu Bar App）

用 `rumps` 库将程序改造为常驻菜单栏的后台应用，彻底告别命令行。

功能点：
- 菜单栏图标实时显示状态（待机 / 录音中 / 转录中）
- 点击图标可切换模式（讯飞 / Whisper）、开关润色
- 开机自启支持（`launchd` plist）
- 右键菜单：查看今日用量、退出

### 4.2 设置界面

轻量设置页（用 `tkinter` 或 webview），替代手动编辑 `.env`：
- API Key 填写与验证
- 触发键自定义（当前硬编码为右 Option）
- 语言偏好、润色开关
- 本地模型选择与下载进度展示

### 4.3 打包分发

将项目打包为独立 `.app`，双击即可运行，无需安装 Python：
- 工具：`py2app` 或 `PyInstaller`
- 目标：拖入 Applications 文件夹即可使用

## Phase 5 — 体验细节

### 5.1 自定义触发键

当前触发键硬编码为右 Option，不够灵活。计划支持通过配置文件指定任意修饰键组合。

### 5.2 历史记录

本地记录每次转录的文本和时间戳，方便查找和复用之前说过的内容。

### 5.3 自定义词汇表

允许用户添加常用词、专业术语，提升特定领域（如代码、人名、品牌）的识别准确率（讯飞和豆包 API 均支持热词功能）。

### 5.4 多语言模式

当前默认中文优先。后续可支持语言快速切换，或自动语言检测模式。

### 5.5 实时状态反馈

当前在语音输入结束后，用户面对的是一个"黑盒"——不知道程序是在正常处理还是卡住了。对标产品的做法：豆包输入法实时流式输出字符；Typeless 显示 "thinking" 进度条，结束后再注入文字。

计划为本工具补充类似的状态提示，候选方案：

| 方案 | 实现难度 | 效果 |
|------|----------|------|
| **系统通知（`osascript` / `pync`）** | 低 | 录音结束时弹出 "转录中…"，完成后弹出结果预览 |
| **菜单栏图标动画** | 中（依赖 Phase 4.1）| 图标实时切换：待机 → 录音中 → 转录中 → 完成 |
| **浮动状态窗口** | 中 | 屏幕角落显示小悬浮框，实时更新状态文字和进度条 |

近期可先用系统通知作为轻量过渡方案，GUI 阶段再升级为菜单栏图标动画。
